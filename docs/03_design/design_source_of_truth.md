# 디자인 Source of Truth 운영 템플릿

## 목적
- 디자인/개발 간 단일 기준 문서를 정의한다.
- Figma Make를 기준으로 핸드오프 일관성을 유지한다.

## 기준 도구
- 1차 기준: `Figma Make` (파일/페이지/노드 ID 명시)
- 2차 기준: `/docs/03_design/*` 문서
- 코드 기준: 구현 PR의 스크린샷 비교 결과

## 운영 규칙 (Figma Make 중심)
- 규칙 1: 시안 변경은 Figma Make에서 먼저 반영한다.
- 규칙 2: 개발 이슈는 노드 ID 단위로 코멘트 링크를 남긴다.
- 규칙 3: 핸드오프 시 `토큰 -> 컴포넌트 -> 화면` 순서로 확정한다.
- 규칙 4: 긴급 패치도 사후에 Figma Make와 동기화한다.

## 토큰/컴포넌트 인수(acceptance) 기준
- 토큰 인수 기준:
- [ ] 네이밍 규칙 준수 (`color.*`, `space.*`, `radius.*`)
- [ ] 라이트 테마 기준 값 문서화
- [ ] 블루/퍼플/라임 팔레트 유지 명시
- 컴포넌트 인수 기준:
- [ ] Variants/States 정의 완료
- [ ] 최소 1개 실사용 화면 연결
- [ ] 접근성 라벨/포커스 상태 정의
