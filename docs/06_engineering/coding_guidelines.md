# 코딩 가이드라인 템플릿

## 목표
- 읽기 쉬운 코드와 예측 가능한 동작을 우선한다.

## 기본 원칙
- 작은 단위 함수 우선
- 명확한 네이밍 사용
- 하드코딩 값 대신 상수/토큰 사용
- 예외 처리 경로를 먼저 정의

## 리뷰 체크리스트
- [ ] 요구사항 충족
- [ ] 에러/엣지 케이스 처리
- [ ] 불필요한 복잡도 제거
- [ ] 로그/모니터링 포인트 반영
- [ ] 테스트 추가/수정

## 금지 항목
- [ ] 의미 없는 주석
- [ ] 사용하지 않는 코드 방치
- [ ] 민감정보 코드 내 하드코딩

## 문서화 규칙
- 공개 함수/모듈에 간단한 사용 예시 작성
- 변경이 큰 경우 `/docs` 동시 업데이트
